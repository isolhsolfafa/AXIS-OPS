# MES 시스템 공수(Man-Hour) 계산 로직 자문 요청

## 배경
제조 현장 MES-lite 시스템(AXIS-OPS)을 개발 중입니다.
모바일 PWA로 작업자가 작업 시작/완료를 찍고, 대시보드(AXIS-VIEW)에서 CT와 공수를 확인하는 구조입니다.

## 현장 작업 패턴
- 제품 1대(SN)에 카테고리별(기구/전장/TM 등) 작업이 있음
- 하나의 카테고리 안에 Task A~D가 존재
- Task A에 작업자 1,2,3이 **동시에** 투입될 수 있음
- A~C 작업은 불규칙적으로 발생하고, D 작업 완료 시 해당 카테고리 종료

## 구체적 예시

```
Task A (UTIL LINE1 작업):

작업자1: 9:00 시작 → 12:00 완료 (3시간)
작업자2: 9:00 시작 → 12:00 완료 (3시간)

휴게시간: 10:00~10:20 (20분)
```

## 두 가지 계산 방식

### 방식 A: CT × 투입 인원
```
CT = 첫시작 ~ 마지막완료 = 3시간
워킹타임 = 3h - 20min(break) = 2h 40m
공수 = 2h40m × 2명 = 5h 20m (MH)
```

### 방식 B: 개인별 실작업시간 합산
```
작업자1: 3h - 20min(break) = 2h 40m
작업자2: 3h - 20min(break) = 2h 40m
공수 = 2h40m + 2h40m = 5h 20m (MH)
```

### 동시 시작/종료 시 → A = B 동일

### 차이가 발생하는 케이스
```
작업자1: 9:00 ~ 11:00 (2시간, break 차감 후 1h40m)
작업자2: 9:00 ~ 12:00 (3시간, break 차감 후 2h40m)

CT = 3h, 워킹타임 = 2h40m

방식 A: 2h40m × 2명 = 5h20m MH → 작업자1이 3시간 일한 것으로 과대 계산
방식 B: 1h40m + 2h40m = 4h20m MH → 실제 기록 기반
```

## 추가 고려사항
- CT(Cycle Time)는 순수하게 "작업 소요 시간"으로, 첫 시작 ~ 마지막 완료 기준
- 출퇴근 인원 + SN별 투입 인원은 DB에서 정확히 확인 가능
- 현장 작업자가 시작/종료 버튼을 정확한 타이밍에 못 누를 가능성 있음
- 휴게시간(오전/점심/오후/석식)은 자동 차감 처리

## 질문
1. 제조 현장 MES에서 공수(Man-Hour) 계산 시 방식 A와 B 중 어느 것이 표준적인가요?
2. 대시보드에서 CT와 MH를 어떤 형태로 표시하는 것이 관리자에게 가장 유용한가요?
3. 작업자마다 시작/종료 시간이 다른 경우, 공수 계산의 업계 관행이 있나요?
4. CT와 MH 외에 추가로 추적해야 할 지표가 있다면 무엇인가요?
